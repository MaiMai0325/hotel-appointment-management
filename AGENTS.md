# AGENTS.md

## Project Overview
このプロジェクトは、民宿・ホテル運営者向けの予約管理システムです。
Airbnb、Expedia、Booking.com、携程、直接予約など、複数の予約経路を一元管理することを目的としています。

## Main Goal
このシステムを「民宿運営専用OS」として育てたいです。
単なる予約表ではなく、予約・清掃・売上・ゲスト対応・定型文管理までまとめて管理できるシステムにしてください。

## Language
- UIは日本語を基本にしてください。
- コード内コメントもできるだけ日本語で書いてください。
- ゲスト向け文面は日本語・英語・中国語に対応できる設計にしてください。

## Design Rules
- 既存のデザインを大きく崩さないでください。
- スマホで見やすいレイアウトを優先してください。
- 管理画面なので、見た目よりも使いやすさを重視してください。
- ボタン、入力欄、一覧表は分かりやすく整理してください。

## Coding Rules
- 既存機能を壊さないでください。
- 大きな変更をする前に、まず現在の構成を確認してください。
- 複雑な実装の前には、先に実装計画を出してください。
- 変更後は、どのファイルを変更したか説明してください。
- 動作確認方法も必ず説明してください。
- エラーが出にくいように、入力チェックを入れてください。

## Reservation Data Fields
予約データには、最低限以下の項目を含めてください。

- id
- 予約サイト
- ゲスト名
- 人数
- 部屋名または部屋番号
- チェックイン日
- チェックアウト日
- 泊数
- 料金
- 支払い状況
- 予約ステータス
- 清掃状況
- 備考
- 作成日
- 更新日

## Booking Source Options
予約サイトの選択肢は以下を基本にしてください。

- Airbnb
- Expedia
- Booking.com
- 携程
- Agoda
- 直接予約
- その他

## Status Options
予約ステータス：

- 確定
- 未確定
- キャンセル
- 変更待ち

支払い状況：

- 未払い
- 支払い済み
- 現地払い
- 返金済み

清掃状況：

- 未清掃
- 清掃予定
- 清掃中
- 清掃済み
- 確認済み

## Priority Features
優先して実装したい機能は以下です。

1. 予約登録フォーム
2. 予約一覧
3. 今日のチェックイン
4. 今日のチェックアウト
5. 今日の清掃予定
6. 月次売上集計
7. 稼働率計算
8. 予約サイト別の売上集計
9. 定型文管理
10. Google Sheets または Firebase 連携

## Data Storage
最初は localStorage で保存して構いません。
ただし、将来的に Google Sheets、Firebase、Supabase などに移行しやすい構造にしてください。

## Important Notes
このシステムは実際の民宿運営で使う予定です。
そのため、見た目だけではなく、実務で使いやすいことを最優先にしてください。
